# Architecture

## Overview

House Hunter v2 is a single-file HTML application (~11,800 lines). All HTML, CSS, and JavaScript live in one file for portability. The app has no server component — everything runs client-side.

## Data Flow

```
HouseSigma Page → Copy/Paste → parseListing() → parseHouseSigma() → parseRooms()
                                                         ↓
                                                  showPreview() → confirmSave()
                                                         ↓
                                                  HOUSES array → localStorage
                                                         ↓
                                              renderAll() → Cards / List / Modal
```

## File Layout (within the single HTML file)

```
<head>
  Google Fonts link
  <style> — All CSS (~620 lines)

<body>
  Header (app title, add button, export button)
  Toolbar (listing count, sort dropdown, view toggle)
  Filter Bar (search, dropdowns for type/style/community/municipality/size/status/factor)
  Main Content (card grid or list view — rendered dynamically)
  Add Panel (slide-over with paste textarea, URL input, parse preview, confirm/cancel)
  Detail Modal (full house detail with metrics, score breakdown, rooms table)

  <script>
    SEED_HOUSES constant (embedded data, replaced on export)
    State & Init (HOUSES array, localStorage load/save, migration)
    Utility functions (formatting, addressing)
    Scoring system (scoreHouse, scoreClass, rankFactor)
    Style configuration (getStyleConfig, getMergedLevel, getDisplayOrder, calcFloorTotals)
    Parser (parseHouseSigma, parseRooms)
    Parse UI (showPreview, showParseError)
    CRUD (confirmSave, deleteHouse, toggleVisited, openAddPanel, closeAddPanel)
    Filtering & Sorting (filteredHouses, sortedHouses, populateFilters, clearFilters)
    Rendering (renderAll, renderCard, renderListRow, openDetail, closeModal)
    Export (exportHTML)
```

## Key Components

### State Management

- `HOUSES` — Global array of house objects. Single source of truth.
- `localStorage` key `house-hunter-v2` — JSON-serialized `HOUSES` array.
- `SEED_HOUSES` — Fallback data embedded in the HTML. Used on first load when localStorage is empty. Replaced with current `HOUSES` data on export.
- `pendingParsed` — Temporary holding variable for a parsed house before user confirms save.
- `editingId` — Set when editing an existing house (re-parse flow). Null for new additions.

### Initialization & Migration

`init()` runs on page load:
1. Loads `HOUSES` from localStorage (or falls back to `SEED_HOUSES`)
2. Runs migrations on all houses:
   - Cleans corrupted room names (feature-word bleed from parser bug)
   - Removes rooms with empty names
   - Recomputes `sharedArea` flags (excludes bedrooms, applies to all levels)
   - Recomputes `floorSqft` via `calcFloorTotals(rooms, style)`
3. Saves migrated data back to localStorage
4. Calls `renderAll()`

### Parser Pipeline

See [PARSER.md](PARSER.md) for full details.

1. **Pre-processing**: Strip noise (image captions, URL slugs), truncate at comparables section
2. **Field extraction**: Regex patterns extract ~20 fields from cleaned text
3. **Room parsing**: Separate regex on original (uncleaned) text extracts room dimensions
4. **Post-processing**: OCR corrections (dimension shifting, metre→feet conversion), shared-area detection
5. **Floor totals**: Style-aware `calcFloorTotals()` computes merged level totals

### Style Configuration System

The `getStyleConfig(style)` function returns a configuration object per house style:

```javascript
{
  mergeMap: { rawLevel → displayLevel },     // how levels merge for display & totals
  scoreLevels: ['main', 'second'],           // which levels count toward scoring
  basementLevels: ['basement'],              // which levels are treated as basement
}
```

See [SCORING.md](SCORING.md) for the full style config table.

Supporting functions:
- `getMergedLevel(rawLevel, style)` — applies mergeMap to a raw room level
- `getDisplayOrder(style)` — returns the ordered list of display levels for a style
- `calcFloorTotals(rooms, style)` — sums room sqft per merged level, excluding shared-area and excluded rooms

### Scoring System

See [SCORING.md](SCORING.md) for full details.

`scoreHouse(h)` returns `{ total: Number, breakdown: Array }`. Two types of factors:
- **Fixed-threshold**: Hard-coded cutoffs (price, tax, type, style, parking, basement, family room, municipality)
- **Pool-ranked**: Compares against all houses in `HOUSES` (lot area, living area, main floor, second floor)

### Rendering

Three view modes, all generated by `renderAll()`:
- **Card Grid** (`renderCard`) — Visual cards with score badge, floor totals, lot dims, tax/mo, tags
- **List View** (`renderListRow`) — Compact table rows
- **Detail Modal** (`openDetail`) — Full detail view with metrics, score breakdown table, rooms table with floor headers and totals

### Filtering & Sorting

`filteredHouses()` chains all active filters:
1. Free text search (matches against street, community, municipality, type, style, basement)
2. Dropdown filters: Property Type, Style, Community, Municipality, Size range, Sold/For Sale status
3. Scoring Factor filter: filters houses that received +/- for a specific scoring factor

`sortedHouses()` sorts filtered results by: Score, Price, Date, Size, Lot, Main Floor (asc/desc).

### Duplicate Detection

On `confirmSave()`, new entries (not edits) are checked against existing houses using `fullAddress()` as a natural key. If a match is found, the user is prompted to update the existing entry. The existing house's `id` and `visited` status are preserved.

### Export

`exportHTML()` reads the current document HTML, replaces the `SEED_HOUSES` constant with the current `HOUSES` data (JSON-serialized), and triggers a download of the complete HTML file. This creates a portable snapshot with all data embedded.
